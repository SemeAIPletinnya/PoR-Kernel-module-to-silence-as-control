# Roadmap

## Місяць 1 — Базова зрозумілість

### README.md
- Додати чітке пояснення концепції (Proof‑of‑Resonance, silence‑as‑control).
- Приклади використання (мінімальний код, як імпортувати й викликати).
- Схема архітектури (можна ASCII‑діаграму чи просту картинку).

### Документація
- Створити базовий гайд: інсталяція, перший запуск, структура проєкту.

### Тести
- Переконатися, що є хоча б базове покриття ключових функцій.

---

## Місяць 2 — Надійність і автоматизація

### CI/CD
- Налаштувати GitHub Actions для автоматичного запуску тестів.
- Додати лінтинг (наприклад, flake8 або black).

### Релізи
- Вести changelog.
- Оформити перший стабільний реліз (v1.0.0-beta).

### Документація
- Додати приклади інтеграції в інші системи.

---

## Місяць 3 — Залучення спільноти

### Issues & Discussions
- Відкрити кілька тем для обговорення (наприклад, “Можливі сценарії застосування”).

### Contributing.md
- Написати правила для контриб’юторів.

### Demo‑проект
- Зробити маленький приклад застосування модуля (наприклад, інтеграція в простий CLI‑інструмент).

### Документація
- Додати FAQ та секцію “Known limitations”.
