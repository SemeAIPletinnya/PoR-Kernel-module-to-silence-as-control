# Contributing

Thanks for taking the time to contribute! This project welcomes bug reports, fixes, and improvements.

## Getting started

1. Fork the repository and create a topic branch from `main`.
2. Make focused changes that are easy to review.
3. Keep commits small and descriptive.

## Development notes

- Keep changes consistent with existing project structure and style.
- Avoid introducing new secrets or tokens into workflows.

## Testing

Run the existing test or validation commands relevant to your change before opening a pull request.

## Pull requests

- Describe the problem and the solution clearly.
- Link any relevant issues.
- Ensure CI checks pass.

## Code of conduct

By participating, you agree to follow the guidelines in `CODE_OF_CONDUCT.md`.
